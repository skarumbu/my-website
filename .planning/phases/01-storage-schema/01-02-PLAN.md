## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Python code → Azurite | Local dev; test fixture creates/destroys test container |
| schema.py → python-frontmatter (yaml.dump) | Library handles YAML serialization; we must pass plain Python types |
| slugs.py → blob container | Reads blob names to check for collisions |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | schema.py: YAML injection via title field | mitigate | set post["title"] = title_string through frontmatter.Post dict API; python-frontmatter calls yaml.dump internally and double-quotes strings containing colons, special chars. Never construct YAML with f-strings or string concatenation. Acceptance criterion: schema.py contains no f"title: {title}" pattern. |
| T-02-02 | Tampering | schema.py: type coercion for published field | mitigate | validate_post() checks isinstance(post.metadata["published"], bool) — rejects string "false" or integer 0. test_published_is_bool verifies this. |
| T-03-01 | Tampering | slugs.py: slug collision overwrites existing post | mitigate | generate_slug() checks existing blob names before returning; appends -2/-3 suffix. test_dedup_suffix and test_dedup_triple verify this. Race condition on concurrent writes documented for Phase 4 (use overwrite=False + catch ResourceExistsError in write API). |
| T-03-02 | Tampering | slugs.py: path traversal via malicious title | mitigate | python-slugify strips all non-alphanumeric chars; output is [a-z0-9-]+ only. Titles like "../../../etc/passwd" produce empty slug → ValueError raised. test_empty_title_raises covers the ValueError path. |
| T-02-03 | Information Disclosure | slugs.py: connection string in error messages | mitigate | RuntimeError message on missing env var does NOT include the connection string value — only states the env var name is missing. |
</threat_model>
