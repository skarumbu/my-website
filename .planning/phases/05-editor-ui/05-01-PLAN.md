## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Azure AD | loginRedirect and acquireTokenSilent cross this boundary; tokens are returned via redirect |
| Browser → posts-api | All fetch calls include Bearer token in Authorization header |
| localStorage → component | Draft data written/read; accessible to any JS on the page (not sensitive) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Spoofing | Write.tsx auth gate | mitigate | useIsAuthenticated() early-return prevents post list rendering; server validates token on GET /api/posts (Phase 4 SEC-02). Client gate is UX; server gate is security. |
| T-05-02 | Elevation of Privilege | authConfig.js redirectUri | mitigate | redirectUri fixed to window.location.origin; Azure AD validates against registered URIs (Pitfall 6 in RESEARCH.md). Pre-deploy manual check required — see checkpoint in Plan 03. |
| T-05-03 | Tampering | postsApiRequest scope | accept | Scope api://825b77cb-1492-406f-9072-923aa536b328/.default grants static permissions only; server validates token claims. Client cannot elevate scope without Azure AD consent. |
| T-05-04 | Information Disclosure | sessionStorage token cache | accept | MSAL uses sessionStorage (storeAuthStateInCookie: false). Tokens expire per AAD policy (~1h). Acceptable per project's established choice (STATE.md Known Pitfalls). |
| T-05-05 | Tampering | DELETE 204 handling | mitigate | Correct: check resp.ok only; do not call .json(). Pattern from Ideas.tsx line 369. Calling .json() on 204 throws SyntaxError — would suppress the success path. |
| T-05-SC | Tampering | npm installs | accept | No new packages installed in this phase. All libraries (msal-react, msal-browser, react-router-dom) previously audited. |
