    useEffect(() => {

          setBody(post.body ?? '');

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Azure AD | acquireTokenSilent and acquireTokenRedirect for all save/load operations |
| Browser → posts-api | POST (create), PUT (update), GET (load) all require Bearer token |
| localStorage → component | Draft data read on mount; written by timer; raw markdown is not sensitive |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-06 | Spoofing | WriteEditor auth gate | mitigate | isAuthenticated() early-return prevents editor rendering; all API calls include Bearer token validated server-side (Phase 4 SEC-02). |
| T-05-07 | Tampering | Token in Authorization header | mitigate | Token acquired via acquireTokenSilent → only sent over HTTPS (Azure SWA). Headers not accessible to other origins (CORS). |
| T-05-08 | Tampering | History stack on first save | mitigate | navigate({ replace: true }) prevents /write/new from remaining in history stack — no back-button looping to blank editor (Pitfall 4 in RESEARCH.md). |
| T-05-09 | Information Disclosure | localStorage raw markdown | accept | Raw markdown is author-authored content (not PII or secrets). localStorage is accessible to any same-origin JS — project has no third-party scripts that would access it. |
| T-05-10 | Tampering | XSS via stored markdown | accept | Editor stores and retrieves raw markdown only. No rendering in editor. Rendering happens in PostReader.tsx via react-markdown (react-createElement, not dangerouslySetInnerHTML). SEC-01 already satisfied. |
| T-05-11 | Elevation of Privilege | getToken redirect loop | mitigate | errorCode 'interaction_in_progress' is caught and re-thrown (not redirected again). Pattern from Ideas.tsx lines 614-623. |
| T-05-SC | Tampering | npm installs | accept | No new packages. All libraries previously audited. |
</threat_model>
