## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-12 | Tampering | DELETE 204 handling in WriteEditor | mitigate | handleDelete checks resp.ok only; never calls resp.json(). Pattern verified against Ideas.tsx. Calling .json() on 204 throws SyntaxError — could mask successful deletes. |
| T-05-13 | Denial of Service | Stale autosave timer | mitigate | Both timer refs (localTimerRef, apiTimerRef) are cleared in the useEffect cleanup return function. No leaked intervals after unmount. |
| T-05-14 | Tampering | API autosave uses interactive token | mitigate | API autosave timer uses getTokenSilent exclusively — never calls acquireTokenRedirect from a background timer. If token expires, timer skips silently. User-triggered Save uses getToken (interactive fallback). |
| T-05-15 | Spoofing | redirectUri after authConfig fix | mitigate | Task 3 checkpoint requires human verification that window.location.origin is registered in Azure AD App Registration before proceeding. AADSTS50011 would prevent login if not registered. |
| T-05-16 | Information Disclosure | Stale closure reading old draft | mitigate | latestDraft ref updated on every field change via useEffect; timer reads latestDraft.current — always current. Prevents autosaving stale (blank) content to API. |
| T-05-SC | Tampering | npm installs | accept | No new packages in this phase. All libraries previously audited. |
