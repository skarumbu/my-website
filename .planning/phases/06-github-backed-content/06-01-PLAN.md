<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| posts-api → GitHub Contents API | Outbound HTTPS call carrying GITHUB_TOKEN; response content is untrusted external input |
| Azure env vars → slugs.py | GITHUB_TOKEN and GITHUB_REPO read from os.environ; not user-supplied but must not be logged |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Information Disclosure | _gh_headers() / GITHUB_TOKEN | mitigate | GITHUB_TOKEN read from os.environ["GITHUB_TOKEN"] only; never logged, never returned in response body; Azure app settings are encrypted at rest |
| T-06-02 | Tampering | GitHub API response content | accept | python-frontmatter parses YAML metadata only — does not execute content; schema.py only reads named fields; no eval/exec of GitHub response |
| T-06-03 | Tampering | slug path traversal (e.g., "../secrets") | mitigate | SLUG_RE = re.compile(r"^[a-z0-9-]+$") in function_app.py validated before any _gh_url() call; _gh_url() interpolates slug directly but SLUG_RE blocks non-slug chars |
| T-06-04 | Spoofing | GitHub 5xx masquerading as content | mitigate | get_file_sha raises RuntimeError on status >= 500; function handlers map exceptions to 502 |
| T-06-SC | Tampering | pip install requests | mitigate | requests is a VERIFIED PyPI package (14+ years, 300M+/week downloads); no slopcheck gate needed — standard library |
</threat_model>
