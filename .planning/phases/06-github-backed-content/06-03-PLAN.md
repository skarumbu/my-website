| Boundary | Description |
|----------|-------------|
| Bicep deploy → Azure Function app settings | GITHUB_TOKEN stored as @secure() param — must not appear in deployment logs or ARM template output |
| posts/*.md frontmatter → parse_post() | Content created locally; validated by python-frontmatter; no external input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-10 | Information Disclosure | githubToken Bicep @secure() param | mitigate | Param marked @secure() — Azure ARM does not log secure param values; PAT value set via portal or az CLI (not hardcoded in Bicep file committed to git) |
| T-06-11 | Elevation of Privilege | GITHUB_TOKEN PAT scope | mitigate | Use fine-grained PAT scoped to skarumbu/my-website with Contents read/write only — no org-level or delete_repo permissions (per RESEARCH.md Security Domain) |
| T-06-12 | Tampering | docs/design/ references post-migration | mitigate | grep for "docs/design" references before deletion (per RESEARCH.md Pitfall 5); update any found |
| T-06-13 | Denial of Service | Bicep removes blobDataContributorRole | accept | Role only granted blob data contributor on the posts storage account; removing it has no impact since posts-api no longer reads/writes blob content |
| T-06-14 | Spoofing | postsContainer removed while AzureWebJobsStorage still uses same account | mitigate | Keep storageAccount + blobService + storageConnectionString in Bicep; only postsContainer resource block removed (per RESEARCH.md Pitfall 6 and PATTERNS.md critical note) |
| T-06-SC | Tampering | pip install on Azure Function deploy | mitigate | requirements.txt removes azure-storage-blob/azure-identity; adds requests — all VERIFIED PyPI packages; no new ASSUMED packages |
</threat_model>
