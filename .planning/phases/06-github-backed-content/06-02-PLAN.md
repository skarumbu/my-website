<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client request → function_app.py write handlers | Untrusted HTTP body and route params; auth gate must fire before any mutation |
| function_app.py → slugs.py GitHub helpers | Internal call; slug validated by SLUG_RE before call |
| GitHub Contents API response → parse_post | External content parsed by python-frontmatter; no code execution |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-05 | Elevation of Privilege | create_post / update_post / delete_post | mitigate | `require_auth(req)` is the first statement in each write handler's try block — no body parse, no slug read, no GitHub call before auth check (ASVS V4 access control) |
| T-06-06 | Tampering | slug path traversal via route param | mitigate | `SLUG_RE = re.compile(r"^[a-z0-9-]+$")` checked in update_post, delete_post, get_post before any _gh_url() call; create_post derives slug from generate_slug() which uses python-slugify output (always safe chars) |
| T-06-07 | Information Disclosure | GITHUB_TOKEN in HTTP response or logs | mitigate | _gh_headers() in slugs.py only — never passed through to response body; function_app.py never reads GITHUB_TOKEN directly |
| T-06-08 | Denial of Service | N+1 list_posts_dir calls | accept | Personal site, <50 posts, 5,000 req/hour GitHub rate limit; N+1 is acceptable at this scale (D-13) |
| T-06-09 | Tampering | GitHub 422 SHA conflict | mitigate | HTTPError with status 422 mapped to HTTP 409 Conflict returned to client; client (editor) can retry |
</threat_model>
