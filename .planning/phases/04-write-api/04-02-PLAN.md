    Call sequence for update_post (must follow this order per D-05/D-06):
      1. try: require_auth(req) / except ValueError: return _unauthorized()
      2. slug = req.route_params.get("slug"); if not slug or not SLUG_RE.match(slug): 400
      3. try: body = req.get_json() / except Exception: 400 Invalid JSON body
      4. try: client = get_container_client(); blob_client = client.get_blob_client(f"{slug}.md")

            original_date = existing.metadata.get("date")
         except ResourceNotFoundError: return 404; except Exception: return 500
      5. Extract title, description, post_body, published from body (same pattern as create_post)
         if not title or not description: return 400
      6. try: post = build_post(title=title, slug=slug, date=original_date, description=description,
                body=post_body, published=bool(published))
            errors = validate_post(post); if errors: return 400 errors[0]
            content = serialize_post(post)
            blob_client.upload_blob(content.encode(), overwrite=True)
            date_val = post.metadata.get("date"); updated_val = post.metadata.get("updatedAt")
            return _json_response({
              "title": post.metadata.get("title"), "slug": post.metadata.get("slug"),

              "updatedAt": updated_val.isoformat() if hasattr(updated_val, "isoformat") else str(updated_val) if updated_val is not None else "",
              "published": post.metadata.get("published"),
            }, status_code=200)

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Internet → Azure Function | PUT/DELETE requests; Easy Auth validates Bearer token and injects X-MS-CLIENT-PRINCIPAL |
| Handler → Blob Storage | DefaultAzureCredential (managed identity); ContainerClient for reads and mutations |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | Elevation of Privilege | update_post handler | mitigate | require_auth(req) is FIRST call; ValueError → _unauthorized() 401; no slug read or storage access before auth check per D-05/D-06 |
| T-04-06 | Elevation of Privilege | delete_post handler | mitigate | require_auth(req) is FIRST call; ValueError → _unauthorized() 401; no delete_blob() called before auth check |
| T-04-07 | Info Disclosure | 401 vs 404 ordering on write routes | mitigate | Auth checked before existence check — unauthenticated request to /api/posts/any-slug always returns 401, never 404; prevents existence leakage |
| T-04-08 | Tampering | slug path traversal (PUT/DELETE /api/posts/../secret) | mitigate | SLUG_RE = r"^[a-z0-9-]+$" rejects any slug with /, ., or other non-safe chars; checked before any blob operation |
| T-04-09 | Tampering | PUT overwriting wrong post via slug collision | mitigate | Explicit download_blob() before upload_blob() — doubles as existence check AND date read; overwrite=True is intentional for the matched blob only |
| T-04-10 | Tampering | YAML injection via title in PUT body | mitigate | serialize_post() uses python-frontmatter; never f-string YAML; same protection as POST |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed this plan |
</threat_model>
